# Notebooks — learn gradient-sankey by building real data projects

Welcome! 👋 If `examples/` is the *"learn the API in six short scripts"* path, this
folder is the *"build real things"* path. Each notebook is a **complete, honest
ETL project** — it goes from a raw public data source all the way to a finished
animated gradient-Sankey, narrating every step (Extract → Transform → Explore →
shape for the chart → render).

Every notebook is **executed and committed with its outputs**, so you can read the
whole story — charts and all — right here on GitHub without running anything.

## How to run them yourself

These notebooks use the **latest** `gradient_sankey.py` in this repo (a few features
are newer than the current PyPI release), so the easiest path is to **clone and run
from the repo root**:

```bash
git clone https://github.com/FG-SC/gradient-sankey.git
cd gradient-sankey
pip install -r requirements.txt        # pandas, requests, matplotlib, …
# Optional, used by some lessons: pip install yfinance odfpy
# FFmpeg (for the MP4 cells only — PNGs work without it):
#   Windows: choco install ffmpeg · macOS: brew install ffmpeg · Linux: sudo apt install ffmpeg
jupyter lab notebooks/
```

Open any notebook and run it top to bottom. They disk-cache their downloads, so a
second run is fast, and they skip the video cells gracefully if FFmpeg is absent.

## What makes a good gradient-Sankey story?

The notebooks keep coming back to the same few principles — worth internalizing:

- **One tidy table.** Everything reduces to `[time, source, target, value]`. Get there and the chart is easy.
- **Conserve the flow.** What enters a node should leave it. When it can't (a "leak" like cost), give the remainder its own honest node.
- **Let the data move.** A *rolling window* or a *ranking race* tells a story a static, ever-growing total can't.
- **Be honest on screen.** Name your approximations (e.g. "this split is company-wide"), reconcile to the official total, and cite the source.
